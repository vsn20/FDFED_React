===============================================
  ELECTROLAND - STEP BY STEP DEMO GUIDE
===============================================

IMPORTANT: Docker Desktop MUST be open and running before any docker commands!


=== STEP 0: OPEN DOCKER DESKTOP ===
- Click Start Menu → Search "Docker Desktop" → Open it
- Wait until it says "Docker Desktop is running" (green icon in taskbar)
- This takes about 30-60 seconds


=== STEP 1: START ALL CONTAINERS (from root folder) ===

    cd C:\PROJECTS\FDFED_USING_REACT\FDFED_React
    docker-compose up --build -d

  This will start 4 containers:
    - electroland-mongodb  (Database)
    - electroland-redis    (Cache)
    - electroland-server   (Backend API)
    - electroland-client   (Frontend Website)

  Wait ~2 minutes for first build. After that it's fast.


=== STEP 2: VERIFY CONTAINERS ARE RUNNING ===

    docker ps

  You should see 4 containers with status "Up" and "(healthy)"


=== STEP 3: SHOW TESTS (open NEW terminal) ===

    cd C:\PROJECTS\FDFED_USING_REACT\FDFED_React\server
    npm test

  Output: 48/48 tests passing ✓

  For coverage report:
    npm run test:coverage

  Then open this file in browser: server\coverage\index.html


=== STEP 4: OPEN THESE URLs IN BROWSER ===

  1. Swagger API Docs:
     http://localhost:5001/api-docs

  2. Redis Performance Report:
     http://localhost:5001/api/performance-report

  3. Search API:
     http://localhost:5001/api/search?q=Samsung

  4. Autocomplete API:
     http://localhost:5001/api/search/autocomplete?q=Sam

  5. Frontend (via Docker):
     http://localhost


=== STEP 5: SHOW REDIS CACHING PROOF ===

  1. Open Chrome DevTools (F12) → Network tab
  2. Visit: http://localhost:5001/api/ourproducts
  3. Click on the request → Headers tab → Look for:
     - X-Cache: MISS  (first time - from MongoDB)
     - X-Response-Time: ~100ms
  4. Refresh the page (F5)
  5. Look again:
     - X-Cache: HIT   (second time - from Redis cache!)
     - X-Response-Time: ~5ms
  This proves 90%+ performance improvement!

=== STEP 7: STOP EVERYTHING WHEN DONE ===

    docker-compose down


===============================================
  TROUBLESHOOTING
===============================================

ERROR: "Docker Desktop is not running" or "pipe not found"
  → Open Docker Desktop app from Start Menu first!

ERROR: "Port 5001 already in use"
  → Close any running "npm start" in other terminals

ERROR: "electroland-server restarting"
  → Run: docker logs electroland-server
  → Check for errors

TO START WITHOUT DOCKER (local development):
  Terminal 1: cd server && npm start
  Terminal 2: cd client && npm run dev
  (Redis won't work locally unless Docker is running)


===============================================
  QUICK COMMANDS CHEAT SHEET
===============================================

  docker-compose up --build -d    Start everything (background)
  docker-compose up -d            Start (no rebuild, faster)
  docker ps                       Show running containers
  docker logs electroland-server  Show server logs
  docker-compose down             Stop everything
  
  cd server && npm test           Run 48 tests
  cd server && npm run test:coverage   Generate coverage report
